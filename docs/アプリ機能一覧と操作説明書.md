# Schedule Manager API アプリ機能一覧と操作説明書

## 1. アプリ概要
このアプリは、個人/フレンド間の**予定（スケジュール）管理**を中心としたWebアプリです。  
主な構成は以下です。

- カレンダーで予定を作成・編集・完了管理
- 予定のフレンド共有、参加申請、承認/却下
- フレンド管理（検索、申請、承認、解除）
- 募集掲示板（スレッド・投稿・興味表明）
- プロフィール編集（画像、自己紹介、X URL、配信URL）
- 通知（アプリ内通知・Web Push・リマインドメール）
- 管理者向けモデレーション（`/admin`）

---

## 2. 画面一覧（URL）

- `/login` ログイン
- `/register` 新規登録
- `/calendar` カレンダー（予定管理のメイン画面）
- `/` カレンダーへ遷移
- `/profile` プロフィール編集
- `/friends` フレンドホーム（ランキング/通知設定など）
- `/friends/search` ユーザー検索・フレンド申請
- `/friends/list` フレンド一覧
- `/friends/requests` 受信したフレンド申請一覧
- `/friends/profile/{username}` ユーザープロフィール表示
- `/board` 募集掲示板
- `/schedules/{id}/join` 参加申請ページ
- `/schedules/{id}/discord-invite` Discord招待URL設定ページ
- `/admin` 管理者モデレーションページ（管理者のみ）
- `/h2-console` H2コンソール（ローカルH2利用時）

---

## 3. 利用者向け機能

## 3.1 認証・アカウント
- フォームログイン/ログアウト
- 新規登録（username, password, email）
- 登録時の `display_name` は `username` と同値で自動設定
- BANされたユーザーは利用不可（管理者が解除可）

## 3.2 予定（スケジュール）管理
- 日別の予定一覧取得
- 月表示用の予定マーカー取得
- 予定の作成/編集/削除
- 完了/未完了切り替え
- タイトル候補の自動提案（過去入力ベース）
- 締切接近リマインダー取得（`windowMinutes` 指定）
- CSVテンプレートダウンロード
- CSVインポート（バリデーションエラー行の返却あり）

### 予定項目の主な属性
- 日付、タイトル、開始/終了時刻
- デバイス種別、ランク帯、説明
- フレンド共有フラグ
- 募集公開フラグ（joinable）
- メッセージ共有可否（messageShareable）
- 募集人数上限（recruitmentLimit）

## 3.3 予定共有・参加申請
- フレンドに共有された予定の閲覧
- 参加申請の送信/キャンセル
- 参加申請時にコメントと任意のゲームIDを送信
- 予定オーナーによる参加申請の承認/却下
- 承認済み参加者のみ再シェア許可（承認ゲート）

## 3.4 Discord招待URL
- 募集予定に `discord_invite_url` を設定/更新
- オーナー・参加者は予定カードから利用可能

## 3.5 フレンド機能
- フレンド検索（username指定）
- フレンド申請送信
- 受信申請の承認
- フレンド解除（2段階確認）
- フレンド通知設定（相手ごとON/OFF）
- フレンドランキング（タスク達成、期間別）

## 3.6 プロフィール機能
- 表示名、メール、自己紹介の更新
- X URL / 配信URLの登録
- アイコン色設定
- プロフィール画像アップロード/削除
- ラベル色（予定ラベル）の個別/全体リセット
- フレンドプロフィール表示で X URL / 配信URL をリンク表示（別タブ）

## 3.7 掲示板（募集）
- `/board` 初期表示は「スレッド作成」「スレッド一覧」の2カード
- スレッド作成カードから `/board/create` へ遷移
- スレッド一覧（キーワード検索）
- スレッド投稿（募集投稿）
- 投稿への参加希望（作成者が了承/見送り）
- 承認メンバー向け Discord 招待URL表示
- 不適切ユーザーの通報

## 3.8 通知
- ナビバッジ（未処理件数）
  - フレンド申請
  - 未読DM
  - 参加申請
- 通知イベント取得（`/api/notifications/events`）
- Web Push 購読/解除
- メールリマインダー（定期ジョブ）

---

## 4. 管理者機能（`/admin`）

`app.admin.usernames` に含まれるユーザーのみ利用できます。

- ユーザー一覧表示
- 対象ユーザー絞り込み（`全ユーザー` 含む）
- 投稿/コメント/DM/予定の横断検索
- 予定削除
- 掲示板投稿削除
- 掲示板興味コメント削除
- DM削除
- ユーザーBAN/解除
- 通報一覧確認・対応済み化

管理APIは `/api/admin/moderation/**` 配下です。

## 6. 開発/運用手順

## 6.1 前提
- Java 21
- Maven Wrapper（`./mvnw`）
- PostgreSQL（通常運用）

## 6.2 起動
```bash
# PostgreSQL 例（ローカル）
export SPRING_DATASOURCE_URL='jdbc:postgresql://localhost:5432/schedule_manager'
export SPRING_DATASOURCE_USERNAME='b-boy49'
export SPRING_DATASOURCE_PASSWORD='Taka3612'
./mvnw spring-boot:run
```

## 6.3 テスト/ビルド
```bash
./mvnw test
./mvnw clean package
./mvnw clean verify
```

## 6.4 Dockerイメージ作成
```bash
docker build -t schedule-manager-api .
```

## 6.5 主要設定（`application.properties`）
- DB接続: `SPRING_DATASOURCE_URL`, `SPRING_DATASOURCE_USERNAME`, `SPRING_DATASOURCE_PASSWORD`
- Flyway: `spring.flyway.*`（マイグレーションは `src/main/resources/db/migration`）
- メール: `MAIL_USERNAME`, `MAIL_APP_PASSWORD`, `MAIL_FROM`
- Web Push: `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`
- 管理者: `app.admin.usernames`（カンマ区切り）

補足:
- ローカル開発の代表設定は H2（`jdbc:h2:file:./data/schedule-db`）
- H2コンソールは `/h2-console`

---

## 7. データベース運用メモ
- スキーマ初期化の基準は Flyway（`V1__init_schema.sql` 以降）
- PostgreSQL運用時は `spring.sql.init` へ依存しない構成
- 主な追加マイグレーション:
  - `V3__add_schedule_discord_invite_url.sql`
  - `V4__add_schedule_join_request_game_id.sql`

---

## 8. トラブルシュート
- ログインできない: ユーザーBAN状態、認証情報、DB接続を確認
- Push通知が届かない: VAPID鍵設定・ブラウザ購読状態を確認
- メール通知が届かない: SMTP認証情報、`app.reminder.enabled`、cron設定を確認
- 参加申請できない: 募集締切、参加上限、既参加状態を確認
- 管理画面が開けない: `app.admin.usernames` にログインユーザー名が含まれるか確認
